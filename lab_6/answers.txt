3. Команда: cd /proc/3736
   Вывод: Переход в папку 3736
   Ответ на вопрос: 
   Они нужны для гибкого управления данными. Благодаря разделению на три канала, мы можем:
   - Получать данные из одного места (клавиатура или файл).
   - Выводить полезный результат в другое (экран или файл).
   - Отправлять сообщения об ошибках в третье (лог-файл), чтобы они не перемешивались с основным результатом.

ЧАСТЬ 2

    1.Команда: echo "Мяу."
    Вывод: Мяу.
    Ответ на вопрос: Потоки вывода (stdout) и ошибок (stderr) можно разделить, потому что в операционной системе Linux они работают через разные файловые дескрипторы:

    - Дескриптор 1 зарезервирован для стандартного вывода (обычные данные).

    - Дескриптор 2 зарезервирован для стандартного потока ошибок (диагностические сообщения).

    Поскольку это два разных «канала» передачи данных, оболочка (bash) позволяет перенаправлять их независимо друг от друга.

ЧАСТЬ 3

    1.Команда: gcc file_demo.c -o file_demo
    Вывод: Сборка программы

    2.Команда: ./file_demo
    Вывод: Файл успешно открыт. Дескриптор: 3
    Данные записаны в myfile.txt. Проверьте файл командой cat.

    3.Команда: cat myfile.txt
    Вывод: Hello, system programming!

    Ответ на вопрос: Это восьмеричное представление прав доступа в Linux (POSIX). Каждая цифра отвечает за свою группу пользователей:
    0 — указывает на то, что это восьмеричное число.
    6 (Владелец) — 4 (чтение) + 2 (запись) = чтение и запись.
    4 (Группа) — только чтение.
    4 (Остальные) — только чтение.

    #define FILELIB_H

    void save_message(const char *filename, const char *message);

    #endif
    --------------------------------------------------
    2.Команда: touch FileLib.c
    Вывод: Создание файла FileLib.c

    #include "FileLib.h"
    #include <fcntl.h>
    #include <unistd.h>
    #include <string.h>

    void save_message(const char *filename, const char *message) {
    int fd = open(filename, O_WRONLY | O_CREAT | O_APPEND, 0644);
    
    if (fd != -1) {
        write(fd, message, strlen(message));
        write(fd, "\n", 1); 
        close(fd);
        }
    }
    --------------------------------------------------
    3.Команда: touch main.c
    Вывод: Создание файла main.c

    #include <stdio.h>
    #include "FileLib.h"

    int main() {
    printf("Это обычный вывод (stdout)\n");

    fprintf(stderr, "Внимание! Сообщение об ошибке (stderr)\n");

    save_message("log.txt", "Программа успешно запущена");
    save_message("log.txt", "Библиотека FileLib работает корректно");

    return 0;
    }
    --------------------------------------------------

    Вывод: Сборка приложение

    Вывод: Это обычный вывод (stdout)

ЧАСТЬ 5

    1.Команда: touch Makefile
    Вывод: создание Makefile

    app: main.c FileLib.c
	gcc main.c FileLib.c -o app

    run: app
	./app

    clean:
	rm -f app log.txt myfile.txt file_demo error.log
    --------------------------------------------------
    2.Команда: make
    Вывод: make: 'app' is up to date.
    --------------------------------------------------
    2.Команда: make run
    Вывод: 
    make: ./app
    Это обычный вывод (stdout)
    Внимание! Сообщение об ошибке (stderr)

    Ответ на вопрос: 
    Он нужен для автоматизации сборки проектов, состоящих из множества файлов. 
    Makefile отслеживает зависимости: если ты изменил только FileLib.c, он пересоберет 
    только его (в больших проектах это экономит часы времени). Также он избавляет разработчика от 
    необходимости помнить сложные флаги компиляции.
